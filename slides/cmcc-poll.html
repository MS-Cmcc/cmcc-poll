Spec first.
Code second.

02 — Stack

Free tier.
No compromises.

Two providers. Zero infrastructure to babysit on event day. If it doesn't run on a free plan, it didn't make the cut.

Framework
Next.js 15
App Router — frontend and API routes in one project. Auto-deploy from GitHub.
✓ Vercel Hobby
Database + Realtime
Supabase
Managed Postgres with built-in Realtime channels. One provider for data and live updates.
✓ Free tier
Schema validation
Zod
Shared schemas across the Markdown parser and API layer. Catches bad configs before going live.
✓ Open source
Question authoring
gray-matter
YAML frontmatter parser for questions.md. Git-versioned. No GUI needed.
✓ Open source
Styling
Tailwind CSS 4
Mobile-first for audience and control panel. Desktop-optimized for the projected display.
✓ Open source
Load testing
k6
200 virtual users, realistic voting behavior, ~6 min run. CLI or GitHub Actions — no infra needed.
✓ Open source
⚡
Key call: only the projected laptop uses Supabase Realtime (1 connection, total). Audience and control panel use 2-second HTTP polling. Result: well under the free tier limit regardless of participant count.
03 — What We Shipped

5 question types.
One file.

Every session is configured by a questions.md in the repo. Edit it, commit it, run a new event. That's it.

🔘
Single Choice
Radio buttons — pick exactly one
☑️
Multiple Choice
Checkboxes — select all that apply
☁️
Word Cloud
Free text — word size scales with frequency
💬
Open-Ended
Free text — answers stream in live as cards
📊
Scale
Parametric slider — custom min/max, mean + distribution
questions.md 
---
type: single_choice
question: How often do you use AI tools?
options:
  - Every day
  - A few times a week
  - Rarely or never
---

---
type: scale
question: How confident are you with AI?
min: 0
max: 10
labels: ["Not at all", "Very confident"]
---

---
type: word_cloud
question: One word for today's session?
max_words: 3
---

---
type: open_ended
question: What would you learn next?
---

---
type: multiple_choice
question: Which AI tools do you use?
options:
  - ChatGPT
  - Claude
  - Gemini
  - Copilot

04 — After the Event

Session ended.
Data's yours.

The moment you close the session, a full results page is waiting. No extra steps. No data wrangling. Open it, export it, done.

Results page — /admin/results/[id]
🔒
Admin-only — protected by ADMIN_CODE
📊
Full breakdown per question — all visualizations, all questions, scrollable
🔗
One tap from the control panel — direct link after session ends
📱
Any device, any browser — just a URL
Export options
📥 CSV — two files
summary.csv — aggregated counts and percentages per question
raw.csv — one row per vote, option labels resolved, ready for analysis
Generated client-side — no server round-trip
🖨 PDF — print to file
Browser-native print dialog. Charts render clean. Word clouds fall back to a frequency table in print mode.
Zero dependencies — window.print() with print CSS
